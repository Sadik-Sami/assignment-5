What is DOM and what is the purpose of DOM?

DOM stands for Document Object Model. It's how a web browser represents a web page internally. The D stands for Document, The O stands for Object, The M stands for Model.
It represents the web page so that programs can change the document structure, style, and content inside of the web page dynamically. It also helps the web page to be more responsive and dynamic. Some basic cases are like adding event listeners to buttons. So that when the user uses the button something functional can happen which will be usefull to the user.
DOM represents the document as nodes and objects or array like objects. That way, programming languages can interact with the page. one of the most popular one is Javascript.

How will you select HTML elements using DOM? Name the DOM methods.

1. document.getElementsByTagName(”Tag Name”);
2. document.getElementsByClassName(”Class Name”);
we will get all the elements with the same class name in an array like object.
3. document.getElementById(”Id name”);
as id's are unique, we will only receive one item
4. document.querySelector(cssSelectors/classNames);
we will need to use (#) for id's and (.) for classes as query selector needs CSS Selectors.
The querySelector() method returns the first child element that matches a specified CSS selector(s) / class names of an element.
5. To get all elements with class="example". querySelectorAll(".example"); The querySelectorAll() method returns a collection of an element's, child elements that match a specified CSS selector(s), as a static Node List object.

What is event bubble?

What is Event Bubble? Event Bubble is a thing of DOM (Document Object Model). It takes place when an element receives an order, and that order bubbles up or we can say that it is propagated to it's parent and parent's parent (ancestor) elements in the DOM tree until it gets to the root element.

What is a callback function and why will you use it?

A callback function is a function passed into another function as an argument or as a parameter. which then takes place in the outer function to complete an action which is needed in the main function (the function where call back function was used).
When we want one function to execute only after another function has completed its work, we use callback functions. It needs to pass as a parameter to other functions to make a function callback just as I explained in the first section.
